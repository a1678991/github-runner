# Packaging (Arch + Nix) ‚Äî Design

**Date:** 2026-06-10
**Status:** Approved (brainstorming phase)
**Companion to:** [2026-06-10-qemu-runner-design.md](2026-06-10-qemu-runner-design.md)

## What this covers

Operator-installable packaging for github-qemu-runner on the two target
ecosystems: an Arch Linux `-git` PKGBUILD (with sysusers/tmpfiles so the
service runs out of the box) and a Nix flake exposing both a package and a
NixOS module. A dedicated GitHub Actions workflow builds both on every
push/PR ‚Äî the PKGBUILD inside the official `archlinux` container image, the
flake via `nix flake check`.

Decisions settled during brainstorming:

| Decision | Choice |
|---|---|
| PKGBUILD model | `-git` VCS package (no releases exist yet); `pkgver()` = `r<count>.<hash>` |
| Nix deliverable | Flake: `buildGoModule` package + NixOS module (`services.github-qemu-runner`) |
| CI | Separate `.github/workflows/packaging.yml`: arch-container makepkg job + nix job |
| License | Add `LICENSE` (MIT) ‚Äî required by PKGBUILD `license=()` and nix `meta.license` |
| Nix formatting | `nixfmt` via mise + lefthook + CI lint (no statix/deadnix ‚Äî YAGNI) |

## Arch packaging ‚Äî `packaging/arch/`

### `PKGBUILD`

- `pkgname=github-qemu-runner-git`, `arch=('x86_64')`, `license=('MIT')`
- `provides=('github-qemu-runner')`, `conflicts=('github-qemu-runner')`
- `source=("git+https://github.com/a1678991/github-qemu-runner.git")`,
  `sha256sums=('SKIP')` (standard for VCS packages)
- `pkgver()`: tagless convention `r<commit-count>.<shorthash>` via
  `git rev-list --count HEAD` + `git rev-parse --short HEAD`
- `depends=(qemu-base cdrtools)` ‚Äî `qemu-system-x86_64`/`qemu-img` and
  `genisoimage` (cdrtools ownership verified on Arch)
- `makedepends=(git go)`; no `checkdepends` (runtime deps cover the gated
  tests; no KVM needed ‚Äî tests use fake binaries)
- `build()`: Arch Go guidelines ‚Äî `-buildmode=pie -trimpath -mod=readonly
  -modcacherw`, external linking flags
- `check()`: `go test ./...` (full suite runs, including qemu-img/genisoimage
  integration-gated tests)
- `package()`:
  - binary ‚Üí `/usr/bin/github-qemu-runner`
  - unit ‚Üí `/usr/lib/systemd/system/github-qemu-runner.service`
  - example config ‚Üí `/etc/github-qemu-runner/config.example.yaml`
  - sysusers/tmpfiles fragments ‚Üí `/usr/lib/{sysusers.d,tmpfiles.d}/`
  - LICENSE ‚Üí `/usr/share/licenses/$pkgname/`, README ‚Üí `/usr/share/doc/$pkgname/`

### `github-qemu-runner.sysusers`

```
u gh-runner - "GitHub Actions QEMU runner" /var/lib/github-qemu-runner
m gh-runner kvm
```

### `github-qemu-runner.tmpfiles`

```
d /var/lib/github-qemu-runner 0750 gh-runner gh-runner -
```

Together these replace the README's manual `useradd`/`mkdir` steps on Arch.
The README gains an Arch section pointing at the package.

### Local test procedure

`makepkg --cleanbuild --syncdeps` in `packaging/arch/`, `namcap` on the
result, `pacman -Qlp` content inspection. (The -git source clones GitHub
main; local test runs after the merge that introduced the code being
packaged, which is satisfied ‚Äî main is merged and pushed.)

## Nix flake ‚Äî root `flake.nix` + `nix/`

### `nix/package.nix`

`buildGoModule`:
- `pname = "github-qemu-runner"`, `version = "0-unstable-<date>"`
- `src = self` (flake source), computed `vendorHash`
- `subPackages = [ "cmd/github-qemu-runner" ]`
- Tests enabled; `nativeCheckInputs = [ qemu cdrkit ]` so the
  qemu-img/genisoimage-gated tests run in the sandbox (nixpkgs' genisoimage
  is in `cdrkit`)
- `meta`: description, homepage, `license = licenses.mit`,
  `platforms = [ "x86_64-linux" ]`, `mainProgram`

### `nix/module.nix` ‚Äî NixOS module

`services.github-qemu-runner`:
- `enable` (bool)
- `package` (defaults to the flake package)
- `settings` (free-form via `pkgs.formats.yaml`; rendered to the config file
  passed via `-config`; same schema as `packaging/config.example.yaml`)
- `privateKeyFile` (path) ‚Äî wired via systemd `LoadCredential`; the module
  sets `settings.github.private_key_path =
  "${CREDENTIALS_DIRECTORY}/app-key.pem"` semantics by default (the module
  controls the unit, so the LoadCredential caveat about manual commands is
  documented in the option description)

Config generated by the module:
- systemd service mirroring `packaging/github-qemu-runner.service` hardening
  (User/Group gh-runner, `SupplementaryGroups=kvm`, `StateDirectory`,
  `ProtectSystem=strict`, `NoNewPrivileges`, `PrivateTmp`,
  `TimeoutStopSec=35m`, `Restart=on-failure`)
- `users.users.gh-runner` system user + `kvm` extra group;
  `users.groups.gh-runner`
- Service `path = [ qemu cdrkit ]` so `qemu-system-x86_64`, `qemu-img`,
  `genisoimage` resolve

### `flake.nix`

- Inputs: `nixpkgs` (nixos-unstable)
- Outputs: `packages.x86_64-linux.default`, `nixosModules.default`,
  `checks.x86_64-linux` = package build + NixOS-module evaluation check
  (a minimal `nixosSystem` that enables the service and asserts it
  evaluates)
- `flake.lock` committed

## CI ‚Äî `.github/workflows/packaging.yml`

Triggers: `push` to `main`, `pull_request`. No paths filter ‚Äî packaging
breakage caused by Go changes is exactly what this must catch.
`permissions: contents: read`; all actions SHA-pinned (pinact); checkout with
`persist-credentials: false`; zizmor-clean.

### Job `arch-package`

`runs-on: ubuntu-latest`, `container: archlinux:base-devel`:
1. `pacman -Syu --noconfirm git go qemu-base cdrtools namcap sudo`
2. checkout; `git config --system safe.directory '*'`
3. Create non-root `builder` user (makepkg refuses to run as root), chown
   the workspace copy
4. **Rewrite the PKGBUILD source to `git+file://$GITHUB_WORKSPACE`** so the
   PR's own code is built (a `-git` PKGBUILD otherwise builds upstream main)
5. `makepkg --noconfirm` as builder (deps pre-installed as root; no
   `--syncdeps` needed)
6. `namcap` the built package (informational: log output; fail only on
   errors, not warnings)
7. Install smoke test: `pacman -U --noconfirm *.pkg.tar.zst`, then
   `github-qemu-runner nonsense` must exit 2, and
   `sysusers`/`tmpfiles` fragments must be present in the package file list
8. Upload the `.pkg.tar.zst` as a build artifact

### Job `nix`

`runs-on: ubuntu-latest`:
1. checkout (`persist-credentials: false`)
2. DeterminateSystems `nix-installer-action` (SHA-pinned)
3. `nix flake check` (builds package, runs go tests in sandbox, evaluates
   the module check)
4. Smoke: `nix run .#default -- nonsense` must exit 2

## Verification (local-first, mirroring CI)

1. `makepkg --cleanbuild --syncdeps` + `namcap` + `pacman -Qlp` locally on
   this Arch host
2. `nix flake check` + `nix build` + `nix run .#default -- nonsense` locally
3. Existing lint matrix stays green (zizmor audits the new workflow;
   nixfmt covers the new .nix files)
4. Push branch ‚Üí packaging.yml runs both jobs green ‚Üí PR

## Out of scope

- AUR publication (the PKGBUILD is AUR-ready; publishing is a later manual
  step)
- Release-pinned (non-git) PKGBUILD ‚Äî revisit when the first version tag is
  cut
- Home Manager module, non-x86_64 platforms, goreleaser
